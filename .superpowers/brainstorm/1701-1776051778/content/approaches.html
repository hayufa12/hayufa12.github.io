Which implementation approach?

All three remove multi-page routing. Differences are in how scroll targeting and active nav state work.

A — Section IDs + scrollIntoView Recommended

Each section gets an id (e.g. id="experience"). Nav click handlers call document.getElementById('experience').scrollIntoView({behavior:'smooth'}). A custom useActiveSection Intersection Observer hook highlights the current nav item as you scroll.

↑ Active link updates as you scroll — no URL change

Pros

  • Simplest code
  • No React Router at all
  • No URL pollution

Cons

  • None significant

B — React refs via Context

Each section exposes a ref. A SectionRefsContext provides refs to the Navbar. Clicks call ref.current.scrollIntoView(). Same Intersection Observer for active state.

Pros

  • More "React" idiomatic
  • No direct DOM access

Cons

  • Extra context boilerplate
  • More files to change
  • Same end result as A

C — Intercept anchor links

Keep <a href="#about"> but call e.preventDefault() + scrollIntoView() to prevent URL hash changes. Keeps anchor semantics but prevents URL update.

Pros

  • Semantic HTML anchors

Cons

  • Fighting browser defaults
  • More fragile
  • Worse than A

Recommendation: A is the cleanest path. Remove React Router entirely — it's no longer needed.
