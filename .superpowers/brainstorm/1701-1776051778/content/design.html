Single-Page Scroll — Design

Here's the full picture of what changes. Scroll down to review each section.

1 · Page structure

App.jsx becomes a flat vertical stack

React Router is removed entirely. All sections render one after another. The Navbar stays sticky at the top.

Before (multi-page)
<BrowserRouter>
<Navbar />
<Routes>
<Route path="/" → Home />
<Route path="/about" → About />
<Route path="/experience" → … />
…
</Routes>
<Footer />
</BrowserRouter>
After (single-page)
<Navbar />
<main>
<Hero /> (from Home.jsx)
<KeyNumbers /> (from Home.jsx)
<About id="about" />
<Experience id="experience" />
<Skills id="skills" />
<Projects id="projects" />
<Contact id="contact" />
</main>
<Footer />
2 · Navbar

NavLink → button with scroll + active via IntersectionObserver

A new useActiveSection hook observes all section IDs and returns whichever is currently in view. The navbar uses this to highlight the active link — same visual result as before, no React Router.

Navbar state as user scrolls to Experience
HF
About Experience Skills Projects Contact
Get in Touch
↑ "Experience" is bold because that section is in view — tracked by IntersectionObserver
